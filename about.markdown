  <h2 style="color:#007bff;font-weight:300;letter-spacing:0.08em;margin:0 0 20px;">Background</h2>
  <p style="color:#aaa;line-height:1.8;margin:0 0 16px;">
    My path into quality engineering started with Java and Selenium in medical device software
    validation — an environment where a test gap is not just a bug report, it is a regulatory
    finding. That context shaped how I think about testing: coverage and traceability are
    engineering discipline, not bureaucracy.
  </p>
  <p style="color:#aaa;line-height:1.8;margin:0 0 40px;">
    Since then I have worked across fintech, retail, and consulting contexts, building automation
    frameworks, leading QA practices on Agile teams, and most recently designing the full testing
    pyramid for client engagements at Deloitte. I hold an AWS Developer Associate, AWS Cloud
    Practitioner, and ISTQB CTFL certification. I work in both French and English.
  </p>

  <h2 style="color:#007bff;font-weight:300;letter-spacing:0.08em;margin:0 0 20px;">My Approach</h2>
  <p style="color:#aaa;line-height:1.8;margin:0 0 16px;">
    Good QA is not about finding bugs after the fact — it is about building the systems that
    prevent bugs from reaching production. I focus on shift-left practices: PR quality gates,
    contract testing, mutation testing that validates test coverage quality, and CI/CD pipelines
    that surface issues at the cheapest possible moment in the delivery cycle.
  </p>
  <p style="color:#aaa;line-height:1.8;margin:0 0 40px;">
    I also believe automation should be owned by the whole team. My role is often to build the
    framework and the tooling that makes it easy for any engineer to write meaningful tests —
    then instrument those tests so they tell you something actionable, not just green or red.
  </p>

  <h2 style="color:#007bff;font-weight:300;letter-spacing:0.08em;margin:0 0 20px;">This Portfolio</h2>
  <p style="color:#aaa;line-height:1.8;margin:0 0 16px;">
    This site is both a personal portfolio and a living QA engineering showcase. It is tested
    like a production application: 11 CI/CD workflows, 7 test frameworks (Cypress, Playwright,
    Robot Framework, Karate, Jest, Pact, StrykerJS), AWS CloudFront deployment, shift-left
    quality gates on every PR, and release readiness checks on every deploy.
  </p>
  <p style="color:#aaa;line-height:1.8;margin:0 0 40px;">
    The goal was to demonstrate the full testing pyramid on a real deployment — not just write
    tests, but build the infrastructure, agents, and dashboards around them.
  </p>
