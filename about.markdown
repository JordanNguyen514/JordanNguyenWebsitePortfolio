  <h2 style="color:#007bff;font-weight:300;letter-spacing:0.08em;margin-bottom:24px;">My Background</h2>
  <p style="color:#aaa;line-height:1.75;margin-bottom:20px;">
    My path into quality engineering started with Java and Selenium in medical device software validation —
    an environment where a test gap isn't just a bug report, it's a regulatory finding. That context
    shaped how I think about testing: coverage and traceability aren't bureaucracy, they're engineering discipline.
  </p>
  <p style="color:#aaa;line-height:1.75;margin-bottom:20px;">
    Since then I've worked across fintech, retail, and consulting contexts, building automation frameworks,
    leading QA practices on Agile teams, and most recently designing the full testing pyramid for client
    engagements at Deloitte. I hold an AWS Developer Associate, AWS Cloud Practitioner, and ISTQB CTFL certification.
  </p>

  <h2 style="color:#007bff;font-weight:300;letter-spacing:0.08em;margin:40px 0 24px;">My Approach</h2>
  <p style="color:#aaa;line-height:1.75;margin-bottom:20px;">
    Good QA isn't about finding bugs — it's about building the systems that prevent bugs from reaching production
    in the first place. I focus on shift-left practices: PR quality gates, static analysis, contract testing,
    and mutation testing that catches gaps in the test suite itself, not just in the product.
  </p>
  <p style="color:#aaa;line-height:1.75;margin-bottom:20px;">
    I believe test automation should be owned by the whole team, not siloed in a QA department.
    My role is often to build the framework and the CI/CD tooling that makes it easy for any engineer
    to write and run tests — then make sure those tests are actually telling you something meaningful.
  </p>

  <h2 style="color:#007bff;font-weight:300;letter-spacing:0.08em;margin:40px 0 24px;">This Portfolio</h2>
  <p style="color:#aaa;line-height:1.75;margin-bottom:20px;">
    This site is both a personal portfolio and a living QA engineering showcase. It's tested like a
    production application: 11 CI/CD workflows, 7 test frameworks, AWS CloudFront deployment,
    and shift-left quality gates on every PR. The goal was to demonstrate the full testing pyramid
    on a real deployment — not just write tests, but build the infrastructure around them.
  </p>
